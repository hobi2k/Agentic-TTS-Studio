# What Is Next.js

## Next.js를 처음 보는 사람에게

Next.js는 React 기반의 웹 프레임워크입니다.

여기서 중요한 단어는 "프레임워크"입니다.

React만 사용할 때는 보통 다음을 직접 정해야 합니다.

- 라우팅
- 빌드 설정
- 서버 연동 방식
- 정적 파일 처리
- 코드 분할 방식

Next.js는 이 기본 틀을 제공합니다.

즉 "React로 앱을 만들 때 자주 필요한 것"을 미리 준비해둔 도구입니다.

## React와 Next.js의 관계

초심자는 자주 이렇게 물어봅니다.

"React랑 Next.js는 뭐가 다른가요?"

쉽게 말하면:

- React는 UI를 만들기 위한 라이브러리
- Next.js는 React 앱을 실제 서비스 형태로 만들기 위한 프레임워크

비유하면:

- React는 엔진 부품
- Next.js는 차의 기본 골격

## 왜 이 프로젝트에 Next.js를 쓰는가

이 프로젝트는 단순 채팅 화면만 필요한 것이 아닙니다.

- 홈페이지
- 채팅 요청 API
- 런타임 상태 API
- 오디오 파일 제공 API
- 문서 페이지

이 모든 것을 한 저장소에서 자연스럽게 다루려면 Next.js가 잘 맞습니다.

## Next.js의 장점

### 1. 파일 기반 라우팅

파일 경로가 URL 구조가 됩니다.

### 2. 서버와 클라이언트 코드를 함께 관리

한 프로젝트에서:

- 브라우저 화면
- 서버 API

를 같이 다룰 수 있습니다.

### 3. 좋은 기본값

초심자가 초기 설정에 시간을 덜 쓰고, 기능 구현에 집중할 수 있습니다.

## 이 프로젝트에서 꼭 기억할 한 줄

이 프로젝트에서 Next.js는 "프런트엔드만 담당하는 도구"가 아니라, 채팅 UI와 서버 API를 함께 운영하는 애플리케이션 프레임워크입니다.
